## 数据流设计

### 情绪识别流程
```
用户输入 → 预处理 → BERT编码 → 情绪分类 → 强度计算 → 风险判定 → 响应生成
```

### 对话评估流程
```
用户消息 → 情绪识别 → 问卷评估 → 风险量化 → 干预推荐 → 响应生成
```

### 危机响应流程
```
高风险检测 → 危机触发 → 资源推送 → 安全计划 → 人工转诊（如需）
```

## 安全架构

### 数据加密
- 用户密码：bcrypt哈希
- 敏感数据：Fernet对称加密（端到端）
- 通信传输：HTTPS/TLS

### 匿名化设计
- 使用匿名ID代替真实身份
- 不存储个人敏感信息
- 数据加密存储

### 合规设计
- 明确的免责声明
- 用户协议与隐私政策
- 符合《个人信息保护法》
